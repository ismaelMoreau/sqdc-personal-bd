Voici un schéma de validation MongoDB basé sur les informations extraites des images fournies. Ce schéma est conçu pour capturer les détails des produits de cannabis, en mettant l'accent sur la richesse des données.

```json
{
  "type": "object",
  "properties": {
    "produit": {
      "type": "object",
      "properties": {
        "nom": { "type": "string" },
        "marque": { "type": "string" },
        "espece": { "type": "string", "enum": ["Indica", "Sativa", "Hybride", "Hybride Sativa"] },
        "format": { "type": "string", "pattern": "^3.5g$" },
        "souche": { "type": "string" },
        "genetique": { "type": "string" },
        "couleur": { "type": "string" },
        "effets_potentiels": { "type": "array", "items": { "type": "string" } },
        "delai_effets": { "type": "string", "pattern": "^90 secondes à 5 minutes$" },
        "partie_plante": { "type": "string" },
        "lieu_culture": { "type": "string" },
        "methode_culture": { "type": "string" },
        "affinage": { "type": "string" },
        "humidite": { "type": "string" },
        "cultive_quebec": { "type": "boolean" },
        "message_avertissement": { "type": "string" }
      },
      "required": [
        "nom",
        "marque",
        "espece",
        "format",
        "souche",
        "effets_potentiels",
        "delai_effets",
        "lieu_culture",
        "methode_culture",
        "cultive_quebec"
      ]
    }
  },
  "required": ["produit"]
}
```

### Explication des champs :
- **nom** : Nom du produit.
- **marque** : Marque du produit.
- **espece** : Type de cannabis (Indica, Sativa, Hybride).
- **format** : Poids du produit (3.5g).
- **souche** : Souche ou variété du produit.
- **genetique** : Informations sur la génétique (si applicable).
- **couleur** : Couleur de la fleur.
- **effets_potentiels** : Liste des effets potentiels.
- **delai_effets** : Délai d'apparition des effets.
- **partie_plante** : Partie de la plante utilisée.
- **lieu_culture** : Lieu où le produit a été cultivé.
- **methode_culture** : Méthode de culture utilisée.
- **affinage** : Détails sur le processus d'affinage.
- **humidite** : Informations sur le contrôle de l'humidité.
- **cultive_quebec** : Indique si le produit est cultivé au Québec.
- **message_avertissement** : Avertissements concernant la consommation.

Ce schéma peut être utilisé pour valider les documents dans une base de données MongoDB, garantissant que toutes les informations pertinentes sont capturées de manière structurée.
