Voici un schéma de validation MongoDB basé sur les informations extraites des images fournies. Ce schéma est conçu pour capturer les détails des produits de haschich, en mettant l'accent sur la richesse des données.

```json
{
  "bsonType": "object",
  "required": ["nom", "format", "variete", "couleur_interne", "couleur_externe", "texture", "methode_extraction", "effets_potentiels", "temps_apparition"],
  "properties": {
    "nom": {
      "bsonType": "string",
      "description": "Nom du produit"
    },
    "format": {
      "bsonType": "double",
      "description": "Poids du produit en grammes"
    },
    "variete": {
      "bsonType": "string",
      "description": "Variété de cannabis utilisée"
    },
    "couleur_interne": {
      "bsonType": "string",
      "description": "Couleur interne du produit"
    },
    "couleur_externe": {
      "bsonType": "string",
      "description": "Couleur externe du produit"
    },
    "texture": {
      "bsonType": "string",
      "description": "Texture du produit"
    },
    "methode_extraction": {
      "bsonType": "string",
      "description": "Méthode d'extraction utilisée"
    },
    "effets_potentiels": {
      "bsonType": "array",
      "items": {
        "bsonType": "string"
      },
      "description": "Liste des effets potentiels"
    },
    "temps_apparition": {
      "bsonType": "string",
      "description": "Temps d'apparition des effets après inhalation"
    },
    "origine": {
      "bsonType": "string",
      "description": "Lieu de culture ou de fabrication"
    },
    "avertissement": {
      "bsonType": "string",
      "description": "Message d'avertissement concernant la consommation"
    }
  }
}
```

### Explication des champs :
- **nom** : Le nom du produit de haschich.
- **format** : Le poids du produit, généralement en grammes.
- **variete** : La variété de cannabis utilisée pour produire le haschich.
- **couleur_interne** : La couleur visible à l'intérieur du produit.
- **couleur_externe** : La couleur visible à l'extérieur du produit.
- **texture** : La texture du produit, indiquant s'il est malléable ou résineux.
- **methode_extraction** : La méthode utilisée pour extraire le haschich (ex. : sans solvant, tamisage à sec).
- **effets_potentiels** : Une liste des effets que le produit peut avoir sur l'utilisateur.
- **temps_apparition** : Le temps nécessaire pour que les effets se manifestent après inhalation.
- **origine** : Le lieu où le produit a été cultivé ou fabriqué.
- **avertissement** : Un message d'avertissement concernant la consommation du produit.

Ce schéma peut être utilisé pour valider les documents dans une base de données MongoDB, garantissant que toutes les informations pertinentes sur les produits de haschich sont correctement enregistrées.
