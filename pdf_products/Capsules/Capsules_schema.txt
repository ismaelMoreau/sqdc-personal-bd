Voici un schéma de validation MongoDB basé sur les informations extraites des images fournies. Ce schéma est conçu pour capturer les détails des produits à base de cannabis, en mettant l'accent sur la richesse des données.

```json
{
  "type": "object",
  "properties": {
    "produit": {
      "type": "string"
    },
    "format": {
      "type": "object",
      "properties": {
        "quantite": {
          "type": "integer"
        },
        "unite": {
          "type": "string"
        },
        "dosage": {
          "type": "number"
        }
      },
      "required": ["quantite", "unite", "dosage"]
    },
    "effets_potentiels": {
      "type": "array",
      "items": {
        "type": "string"
      }
    },
    "temps_appartion_effets": {
      "type": "string"
    },
    "ingredients": {
      "type": "array",
      "items": {
        "type": "string"
      }
    },
    "methode_extraction": {
      "type": "string"
    },
    "producteur": {
      "type": "string"
    },
    "fabrique_a": {
      "type": "string"
    },
    "mise_en_garde": {
      "type": "string"
    }
  },
  "required": [
    "produit",
    "format",
    "effets_potentiels",
    "temps_appartion_effets",
    "ingredients",
    "methode_extraction",
    "producteur",
    "fabrique_a"
  ]
}
```

### Explication des champs :

- **produit** : Nom du produit.
- **format** : Détails sur le format du produit, incluant la quantité, l'unité (ex. capsules, mg) et le dosage par unité.
- **effets_potentiels** : Liste des effets potentiels associés au produit.
- **temps_appartion_effets** : Temps estimé pour que les effets se manifestent après ingestion.
- **ingredients** : Liste des ingrédients contenus dans le produit.
- **methode_extraction** : Méthode utilisée pour extraire les composants actifs du cannabis.
- **producteur** : Nom du producteur du produit.
- **fabrique_a** : Lieu de fabrication du produit.
- **mise_en_garde** : Avertissements ou précautions à prendre lors de la consommation du produit.

Ce schéma peut être utilisé pour structurer les données dans une base de données MongoDB, facilitant ainsi la gestion et la recherche d'informations sur les produits à base de cannabis.
