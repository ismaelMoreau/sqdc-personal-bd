Voici un schéma de validation MongoDB basé sur les informations extraites des images fournies. Ce schéma est conçu pour capturer une variété de détails sur les produits de cannabis, en mettant l'accent sur la richesse des données.

```json
{
  "type": "object",
  "properties": {
    "produit": {
      "type": "string"
    },
    "marque": {
      "type": "string"
    },
    "dominance": {
      "type": "string",
      "enum": ["Sativa", "Indica", "Hybride"]
    },
    "terpenes": {
      "type": "string",
      "pattern": "^[0-9]+(\\.[0-9]+)?%-[0-9]+(\\.[0-9]+)?%$"
    },
    "culture": {
      "type": "object",
      "properties": {
        "lieu": {
          "type": "string"
        },
        "méthode": {
          "type": "string"
        },
        "type_de_mouture": {
          "type": "string"
        },
        "affinage": {
          "type": "string"
        }
      },
      "required": ["lieu", "méthode"]
    },
    "emballage": {
      "type": "object",
      "properties": {
        "type": {
          "type": "string"
        },
        "description": {
          "type": "string"
        }
      }
    },
    "effets": {
      "type": "array",
      "items": {
        "type": "string"
      }
    },
    "temps_apparition_effets": {
      "type": "string",
      "pattern": "^90 secondes à 5 minutes$"
    },
    "autres_informations": {
      "type": "string"
    }
  },
  "required": ["produit", "marque", "dominance", "culture", "emballage", "effets", "temps_apparition_effets"]
}
```

### Explication des champs :

- **produit** : Nom du produit.
- **marque** : Marque du produit.
- **dominance** : Type de dominance (Sativa, Indica, Hybride).
- **terpenes** : Plage de terpènes, formaté comme "1.5%-4.9%".
- **culture** : Détails sur la culture, y compris le lieu, la méthode et le type de mouture.
- **emballage** : Informations sur le type d'emballage et sa description.
- **effets** : Liste des effets potentiels du produit.
- **temps_apparition_effets** : Temps nécessaire pour ressentir les effets.
- **autres_informations** : Informations supplémentaires pertinentes.

Ce schéma peut être utilisé pour valider les documents dans une base de données MongoDB, garantissant que toutes les informations pertinentes sur les produits de cannabis sont correctement structurées et stockées.
