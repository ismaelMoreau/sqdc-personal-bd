Voici un schéma de validation MongoDB basé sur les informations extraites des images fournies. Ce schéma est conçu pour capturer les détails des produits de cannabis, en mettant l'accent sur la richesse des données.

```json
{
  "type": "object",
  "properties": {
    "produit": {
      "type": "string"
    },
    "format": {
      "type": "string"
    },
    "espèce": {
      "type": "string",
      "enum": ["Indica", "Sativa", "Hybride", "Mélange"]
    },
    "variété": {
      "type": "string"
    },
    "génétique": {
      "type": "string"
    },
    "lieu_de_culture": {
      "type": "string"
    },
    "méthode_de_culture": {
      "type": "string"
    },
    "méthode_d_extraction": {
      "type": "string"
    },
    "couleur_du_produit": {
      "type": "string"
    },
    "texture": {
      "type": "string"
    },
    "emballage": {
      "type": "string"
    },
    "effets_potentiels": {
      "type": "array",
      "items": {
        "type": "string"
      }
    },
    "temps_d_appartion_effets": {
      "type": "string"
    },
    "producteur": {
      "type": "object",
      "properties": {
        "nom": {
          "type": "string"
        },
        "adresse": {
          "type": "string"
        }
      },
      "required": ["nom"]
    }
  },
  "required": [
    "produit",
    "format",
    "espèce",
    "variété",
    "lieu_de_culture",
    "méthode_de_culture",
    "effets_potentiels",
    "temps_d_appartion_effets"
  ]
}
```

### Explication des champs :
- **produit** : Nom du produit.
- **format** : Détails sur le format (ex. : 3 x 0.5 g).
- **espèce** : Type de cannabis (Indica, Sativa, Hybride, Mélange).
- **variété** : Spécificité de la souche.
- **génétique** : Parenté génétique de la souche.
- **lieu_de_culture** : Localisation de la culture.
- **méthode_de_culture** : Méthode utilisée pour cultiver le cannabis.
- **méthode_d_extraction** : Technique d'extraction utilisée.
- **couleur_du_produit** : Couleur du produit final.
- **texture** : Texture du produit.
- **emballage** : Informations sur l'emballage.
- **effets_potentiels** : Liste des effets potentiels.
- **temps_d_appartion_effets** : Délai d'apparition des effets après inhalation.
- **producteur** : Informations sur le producteur, incluant le nom et l'adresse.

Ce schéma peut être utilisé pour valider les documents dans une base de données MongoDB, garantissant que toutes les informations pertinentes sont capturées de manière structurée.
